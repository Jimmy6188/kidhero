## 代码审查规则

### 自动审查（每次写完代码后必须执行）

在使用 Write 或 Edit 工具修改代码后，**必须立即进行快速审查**：

1. **检查类型安全**：TypeScript 类型是否正确，是否有 any 滥用
2. **检查错误处理**：是否缺少 try-catch，是否有未处理的异常
3. **检查边界条件**：空值、数组越界、除零等
4. **检查安全问题**：认证、权限、SQL 注入、XSS
5. **检查性能问题**：N+1 查询、冗余操作、内存泄漏

### 审查输出格式

发现问题时，使用以下格式报告：

```
===== 代码审查 =====
[High] 文件:行号 - 问题描述
[Medium] 文件:行号 - 问题描述
[Low] 文件:行号 - 问题描述
====================
```

### 审查后处理

- **High 级别**：立即修复
- **Medium 级别**：询问用户是否修复
- **Low 级别**：记录但不阻塞

### 深度审查（手动触发）

如需更全面的审查，可手动运行：
```bash
ocr review
```
